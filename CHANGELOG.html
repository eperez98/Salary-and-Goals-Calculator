v2.0.5 — Stability Patch ✓ Current April 10, 2026
Stability patch for the v2.0 Modern Edition. Resolves all known launch failures on Python 3.14 + Windows when running the .py file directly. The EXE build was unaffected by these issues.
🐛 Bug Fixes
  • FIX_tkinter.TclError: image "pyimage1" doesn't exist — PhotoImage objects created during the language boot screen and splash were cached in _icon_cache. When those widgets were destroyed, Tk invalidated the underlying image data, but the Python objects remained in the cache. CalcApp._build_chrome() then retrieved dead references. Fixed by calling _icon_cache.clear() after each screen teardown, before CalcApp builds its own widget tree.
  • FIXModuleNotFoundError: No module named 'customtkinter' — Running python SalaryGoals_Modern.py failed if dependencies were not installed. Added _ensure_deps() which runs before any imports, detects missing packages with importlib.util.find_spec(), shows a dialog, and installs them automatically via pip.
  • FIXAttributeError: module 'importlib' has no attribute 'util' — In Python 3.14, import importlib does not automatically load the util submodule. Must be imported explicitly as import importlib.util.
  • FIXCrash log never generated — The previous crash logger lived inside if __name__ == "__main__":, which was never reached when the crash happened at import time. Replaced with a global sys.excepthook + sys.stderr redirect installed at line 1, before any other import.
🛡 Reliability Improvements
  • NEWGlobal sys.excepthook installed at line 1 — catches crashes at module import time, not just inside __main__
  • NEWsys.stderr redirected to SalaryGoals_crash.log immediately — captures C-extension error messages that bypass Python's exception system
  • NEW_ensure_deps() — auto-installs customtkinter, pillow, reportlab on first run with a user-facing dialog
  • NEWCrash log always written next to the running script or EXE, using sys.frozen to detect PyInstaller context
  • CHANGEexcept Exception → except BaseException in __main__ to catch SystemExit and KeyboardInterrupt

Cancelled Versions
v1.1.1 — Planned Hotfix ⊘ Cancelled Never shipped
⊘ Why this version was cancelled v1.1.1 was started as a hotfix for two bugs found right after v1.1 shipped: a launch crash (AttributeError: _step_resultado) and the language popup not centering on screen. The fixes were developed and validated, but rather than shipping them as a patch on top of the aging plain-tkinter architecture, the decision was made to skip v1.1.1 entirely and fold all fixes directly into the v2.0 Modern Edition rewrite. Both bugs are fully resolved in v2.0.
  • FIXWas going to fix: AttributeError: 'CalcApp' has no attribute '_step_resultado' — def declaration was dropped during v1.1 chart injection. Resolved in v2.0 full rewrite.
  • FIXWas going to fix: Language popup not centered — winfo_screenwidth() called before update_idletasks(). Resolved in v2.0 with screen dims cached before any UI.
v0.3 Beta ⊘ Cancelled Never shipped
⊘ Why this version was cancelled v0.3 Beta was underway when repeated Python 3.14 crashes from creating Toplevel before root.update() made it unshippable. All completed work (Mexico taxes, canvas flags, Win11 themes, animated splash) was frozen and rebuilt from scratch as v1.0 RC.
